# Phase 27.7 Plan 03: Lint Rules + Static Tests Summary

Static linter (`lintMcpToolsDir`) ships at `scripts/lib/mcp-tools-lint/`, enforcing 4 thin-wrapper invariants against the gdd-mcp tools directory. Phase 27.7 success criteria #4 (zero write tools) and #5 (no `fs.*`/`path.*` in tools) become structurally guaranteed: if a future contributor adds a write tool, imports `node:fs` into a tool file, exceeds 30 LOC, or pushes the count past 12, `tests/gdd-mcp-tools-lint.test.cjs` fails the build.

## What Shipped

### `scripts/lib/mcp-tools-lint/index.cjs`

`lintMcpToolsDir({dir, maxLoc?, toolCap?, exemptions?})` returns `{ violations: LintViolation[], summary: { files_scanned, violations_count } }`. Four rules wired:

| Rule              | Origin | What it scans                                                                                       |
| ----------------- | ------ | --------------------------------------------------------------------------------------------------- |
| `forbid-fs-path`  | D-06   | Line-by-line regex match for `from 'node:fs'`, `from 'fs'`, `from 'node:path'`, `from 'path'`, plus CommonJS `require()` variants. Exempt: `index.ts`, `shared.ts`. |
| `max-loc`         | D-06   | Non-blank-non-comment line count > 30. Skips blank lines, `//` comments, `/*`/`*` block comments. Exempt: `index.ts`, `shared.ts`. |
| `no-write-names`  | D-04   | Extracts `export const name = 'ŌĆ”'` and matches against `/_(create\|update\|delete\|append\|clear\|write\|set)(?:_\|$)/`. Applies to ALL `.ts` files (write-named symbol in `shared.ts` would also be a bug). |
| `tool-count-cap`  | D-03   | Counts files matching `^gdd_[a-z0-9_]+\.ts$` glob. Violation when count > 12. |

Public exports: `lintMcpToolsDir`, `RULES`, `DEFAULT_EXEMPTIONS`, `DEFAULT_MAX_LOC`, `DEFAULT_TOOL_CAP`, `FORBIDDEN_IMPORT_PATTERNS`, `WRITE_NAME_PATTERN`.

### `scripts/lib/mcp-tools-lint/index.d.cts`

Ambient declarations for the `.cjs` module: `LintViolation`, `LintSummary`, `LintResult`, `LintOptions` interfaces + typed `lintMcpToolsDir`, `RULES`, `DEFAULT_EXEMPTIONS`, `DEFAULT_MAX_LOC`, `DEFAULT_TOOL_CAP`, `FORBIDDEN_IMPORT_PATTERNS`, `WRITE_NAME_PATTERN` constants. Phase 27.6 convention: any `.cjs` lib that *could* be imported from `.ts` ships a sibling `.d.cts`.

### `tests/gdd-mcp-tools-lint.test.cjs`

10 tests inside the `27.7-03: gdd-mcp tools lint` describe block:

1. **production-clean**: scans actual `scripts/mcp-servers/gdd-mcp/tools/` ŌĆö asserts `violations.length === 0` and `files_scanned >= 12`. This is the canary; if any of Plans 27.7-04..07 introduce a regression, this fails.
2. **forbid-fs-path (node:fs)**: synthetic `gdd_bad.ts` with `import { readFileSync } from 'node:fs'` ŌåÆ flagged.
3. **forbid-fs-path (bare `path`)**: synthetic `gdd_bad2.ts` with `import { join } from 'path'` ŌåÆ flagged.
4. **no-write-names (gdd_decision_append)**: synthetic tool file with that exact name ŌåÆ flagged.
5. **no-write-names (all 7 verbs)**: parametrized ŌĆö 6 verbs (create/update/delete/write/set/clear) tested in one loop, all flagged. (`append` covered in test 4.)

## Decisions Made

1. **`require(.../index.cjs)` explicit path** ŌĆö Node's directory resolution does NOT auto-pick `index.cjs` without a `package.json` `main` field; the repo's existing convention (matches `tests/audit-aggregator.test.cjs`, `tests/phase-23-baseline.test.cjs`) is to spell out `/index.cjs`. Initial test draft used implicit dir-resolution, surfaced module-not-found at REDŌåÆGREEN transition, fixed in same commit.
2. **WRITE_NAME_PATTERN trailing-token boundary** ŌĆö `/_(create|update|delete|append|clear|write|set)(?:_|$)/` requires `_` before the verb and either `_` or end-of-string after. This correctly flags `gdd_decision_append` but does NOT false-positive on hypothetical names like `gdd_appendix_list` (where `append` is a substring but not a trailing token). Tested against 7 synthetic write-verb names; all flagged.
3. **`extractToolName` regex is type-annotation-tolerant** ŌĆö uses `/export\s+const\s+name\s*(?::\s*[A-Za-z<>{}\s,|]+)?\s*=\s*['"]([^'"]+)['"]/` so `export const name: string = '...'` (currently unused but possible) doesn't escape the lint.
4. **Rule C applies to exempt files too** ŌĆö a write-named symbol in `index.ts` or `shared.ts` would itself be a bug (you should not even *define* a write-flavored exported name in helper modules). Rule A and Rule B respect exemptions; Rule C does not.
5. **`countLoc` strips block-comment opener-and-continuation lines** (`/*`, `*`) ŌĆö covers the common `/** ŌĆ” */` JSDoc style without requiring a full comment parser. False-negatives (e.g. a line of code that *also* starts with `*` after trimming) are impossible since `*` is not a valid statement starter in TS.
6. **`exemptions` is a `Set` (not an array)** ŌĆö O(1) lookups during the scan loop; matches the function signature in the plan spec.
