## What shipped

### 1. `reference/debug-feedback-loops.md` filled (64 -> 119 lines)

Replaced the 28.5-07 scaffold's placeholder section with the full mattpocock-adapted catalog:

- **10 construction paths in priority order**, each with "when to reach for it" trigger and shape:
  1. Failing test — strongest signal; native test framework
  2. `curl` against running endpoint — HTTP, jq assertions
  3. CLI fixture — bash one-liner over fixture input
  4. Headless browser — Playwright/Puppeteer with deterministic seed
  5. Trace replay — `rr` / record-and-replay
  6. Throwaway harness — 10-line `main()` side-stepping framework setup
  7. Fuzz — property-based / AFL / libfuzzer
  8. Bisect — `git bisect run <script>`, O(log N)
  9. Differential — two implementations, output diff
  10. HITL bash — last resort, human reads screen
- **Iterate-on-loop discipline**: cache setup, narrow scope, pin time, seed RNG, isolate filesystem, freeze network — with concrete tool examples per axis (`sinon.useFakeTimers`, `pytest -k`, `nock`, `tc qdisc add dev lo root netem`, etc.).
- **Non-deterministic-bug branch**: measure baseline rate, raise stressors, target the suspect axis, "30% reproduction rate is debuggable; 5% isn't."
- **"When the loop is good enough" gate**: < 1 minute, deterministic (or >= 30% non-determ), automatable, fresh-agent-runnable.

### Test-precision refinement

**3. [Rule 3 - Blocking] Loose regex in plan's Test 8 sample produces unreliable ordering signal**
- **Found during:** Task 3 authoring
- **Issue:** The plan's `<action>` sample test code for Test 8 used `/Hypothesis|Investigation/i.test(lines[i])` to find the hypothesis step. This pattern matches the word "investigation" or "hypothesis" anywhere in the file — including prose paragraphs, frontmatter descriptions, and even the Phase 1 step itself (which contains "hypothesizing"). The same Phase 1 line would match BOTH the Phase 1 regex AND the Hypothesis regex.
- **Fix:** Wrote the ordering test using an anchored numbered-bold step-header regex (`^\\d+\\.\\s+\\*\\*([^*]+)\\*\\*`). The check now extracts the step label only (between `**...**`) and matches against that. Locks structural step ordering rather than loose word presence.
- **Files modified:** tests/debug-feedback-loop.test.cjs (not the plan-sample's literal copy; refined version)
