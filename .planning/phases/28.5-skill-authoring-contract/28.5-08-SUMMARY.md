### Hook extension (`hooks/gdd-decision-injector.js`)

Added 4 helper functions + 2 block-builders + main() wire-up:

- `readContextMd(repoRoot)` — splits CONTEXT.md on `^## ` headings; parses each term + body + optional `**Aliases:**` line.
- `readAdrs(repoRoot)` — walks `docs/adr/`, filters to `NNNN-*.md` files, extracts title from frontmatter `title:` or first H1.
- `matchContextEntries(entries, tokens)` — two-way substring match (term + aliases vs file tokens, case-insensitive).
- `matchAdrEntries(entries, tokens)` — title-token substring match.
- `buildGlossaryBlock()` / `buildAdrBlock()` — bounded output (top-N=5, 200-char body cap).

All file reads defensive: missing/malformed files yield `[]` silently. Output appended AFTER existing recall + prototyping blocks. Hook stays idempotent + deterministic (no globals). Module still loads cleanly; all 12 existing decision-injector tests pass.
