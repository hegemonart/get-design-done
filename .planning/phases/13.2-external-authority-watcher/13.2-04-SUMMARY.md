## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking Issue] `plugin.schema.json` + `marketplace.schema.json` version regex rejected the locked 1.0.7.2**

- **Found during:** Task 3 verification, first `node scripts/validate-schemas.cjs` run after the version bump.
- **Issue:** Both schemas enforce `version: { pattern: "^\\d+\\.\\d+\\.\\d+$" }` — a strict 3-segment SemVer match. The locked CONTEXT §D-29 version `1.0.7.2` is a 4-segment off-cadence decimal patch (matching the existing `1.0.4.1` precedent in CHANGELOG.md). The validator reported two FAIL pairs: `plugin via ajv-validate` on `/version` and `marketplace via ajv-validate` on `/metadata/version`. Without the fix, `npm run validate:schemas` — a top-level CI gate and a Task 3 acceptance criterion — would have gone red on this PR.
- **Fix:** Relaxed the regex on all three `version` fields (plugin.json, marketplace.metadata.version, marketplace.plugins[].version) from `^\\d+\\.\\d+\\.\\d+$` to `^\\d+\\.\\d+\\.\\d+(\\.\\d+)?$`. Backward-compatible — every existing 3-segment version continues to match. Still rejects malformed or unbounded-segment input.
- **Files modified:** `reference/schemas/plugin.schema.json`, `reference/schemas/marketplace.schema.json`.
- **Verification:** `node scripts/validate-schemas.cjs` → 6 pairs OK, 0 failures. `node scripts/verify-version-sync.cjs` → all manifests at 1.0.7.2.
- **Committed in:** `6f0b2ec` (bundled with Task 3 version-bump commit).

**2. [Rule 3 — Blocking Issue] `tests/semver-compare.test.cjs` EXPECTED_SEQUENCE rejected 1.0.7.2**

- **Found during:** Task 3 verification, `npm test` sweep after the version bump + schema relaxation.
- **Issue:** The test's `EXPECTED_SEQUENCE` array lists only 3-segment strings (`'1.0.0'` … `'1.0.7'`) and the sequence-membership assertion requires `plugin.json.version` to be in that array. The locked `1.0.7.2` was absent from the array. Adding `1.0.7.2` directly to `EXPECTED_SEQUENCE` would have broken the sibling `isExactPatchBump` invariant (which requires `b.patch === a.patch + 1` — a 3-segment-only comparison). The plan cannot ship without fixing this CI gate.
- **Fix:** Introduced an `OFF_CADENCE_VERSIONS` Set containing `'1.0.7.2'` and relaxed the "version is in expected sequence" assertion to accept either a mainline `EXPECTED_SEQUENCE` member or a recognized off-cadence version. The consecutive-patch-bump invariant is unchanged — it still only iterates `EXPECTED_SEQUENCE`. Off-cadence versions are deliberately excluded from that check because they are branches off an already-shipped parent, not a new mainline step. Comments in the file explain the carve-out rationale.
- **Files modified:** `tests/semver-compare.test.cjs`.
- **Verification:** `npm test` → 233 pass / 0 fail / 1 skipped.
- **Committed in:** `6f0b2ec` (bundled with Task 3).
