# Privacy Policy

**Effective date:** 2026-04-21
**Plugin:** Get Design Done (`get-design-done`)
**Repository:** https://github.com/hegemonart/get-design-done

## Summary

Get Design Done is a Claude Code plugin that runs locally inside your Claude Code session. It does not operate a server, does not collect telemetry, and does not transmit your data to the plugin author. This policy describes what the plugin reads, writes, and — only when you explicitly trigger it — sends to third-party services you have already connected.

## What the plugin does locally

When you invoke its commands, the plugin reads from and writes to files inside your project directory, including:

- `.design/` — pipeline artifacts (BRIEF.md, DESIGN-CONTEXT.md, DESIGN-PLAN.md, verification reports, cycle archives, sketches, spikes)
- `.design/telemetry/` — local cost and timing logs in JSONL format
- `.design/cache-manifest.json`, `.design/agent-metrics.json`, `.design/budget.json` — local caches and budgets
- `./.claude/skills/` — project-scoped skills generated by the pipeline
- Your source code — only the files the plugin is asked to read, plan, or modify

All of the above stays on your machine. None of it is uploaded by the plugin itself.

## What the plugin sends over the network

The plugin makes network calls **only** when you explicitly invoke a command that requires one. The calls it can make are:

1. **Anthropic API** — every Claude Code session already communicates with Anthropic to run the model. The plugin does not add or bypass this; it uses the model you are already running. Your prompts and file contents included in those prompts are governed by Anthropic's own privacy policy.
2. **Figma MCP** (`get-design-done:figma-*`, connection: `figma.md`) — reads Figma metadata and writes annotations, tokens, and Code Connect mappings to files you specify. Only runs when you invoke a Figma command and have the Figma MCP server connected.
3. **Design authority feeds** (`/gdd:watch-authorities`) — fetches a curated list of public design RSS/Atom feeds listed in `reference/authority-feeds.md`. No credentials, no identifying headers beyond a standard user agent.
4. **GitHub releases** (`/gdd:check-update`) — checks the plugin's own GitHub releases endpoint for new versions. A 24-hour cache is kept in `.design/update-cache.json`.
5. **Third-party tool connections** you choose to enable via `/gdd:connections` — such as 21st.dev Magic MCP, Magic Patterns, Chromatic, Storybook, Graphify, Pencil.dev, Paper.design, Refero, Preview, Pinterest. Each is opt-in, configured by you, and governed by its own privacy policy. The plugin does not proxy traffic through the author's infrastructure.
6. **`WebFetch` / `WebSearch`** — only when an agent is explicitly configured to research and you approve the tool call.

The plugin has no analytics, no crash reporting, and no "phone home" behavior.

## Secrets and credentials

Any API keys or tokens for third-party services (Figma, Chromatic, etc.) are configured through your Claude Code environment or MCP server settings, not through the plugin. The plugin reads them only insofar as the underlying tool passes them through. Telemetry written to `.design/telemetry/` includes a secret scrubber pass designed to strip common credential patterns; nevertheless you should treat `.design/` as containing project-sensitive material and gitignore it if appropriate.

## Data retention and deletion

All plugin-generated data lives in your project directory. You can delete it at any time with `rm -rf .design/` (or the equivalent). There is nothing to request deletion of from the plugin author, because nothing is stored on our side.
